# Bécane Paris → 의뢰소 기능 매핑 분석

## 1. 현재 구현 상태 평가

### 문제점

| 페이지 | 현재 상태 | 문제 |
|--------|----------|------|
| 랜딩 (`_index.tsx`) | 어두운 배경 + 빨간 CTA | Bécane 스타일과 불일치, 여백 부족 |
| 음악 선택 (`music.tsx`) | 세로 리스트 + 빨간 테두리 | 일반적인 앱 UI, 미니멀하지 않음 |
| 멤버 선택 (`member.tsx`) | 3열 그리드 + 동그란 아바타 | Bécane의 Chronophotography 미적용 |
| 모션 선택 (`motion.tsx`) | 3열 그리드 + 이모지 | 썸네일/GIF 없음, 카드 디자인 촌스러움 |

### 공통 문제
- ❌ 여백이 부족함 (Bécane: 섹션 간 80px+)
- ❌ 컬러가 산만함 (빨강 CTA가 과함)
- ❌ 타이포그래피가 밋밋함 (Bold 대형 타이틀 없음)
- ❌ 콘텐츠(이미지/비디오) 중심이 아님
- ❌ 플로팅 바 패턴 미적용

---

## 2. Bécane → 의뢰소 페이지 매핑

```
┌─────────────────────────────────────────────────────────────────┐
│  Bécane Paris              →           의뢰소 (Request Lab)     │
├─────────────────────────────────────────────────────────────────┤
│  메인 (Chronophotography)  →  랜딩 + 멤버 선택 (하이브리드)      │
│  ALL 상품 그리드           →  모션 선택                         │
│  Look 상세                 →  결과 페이지                       │
│  상품 상세                 →  (해당없음)                        │
│  Stories                   →  음악 선택 (갤러리 스타일)          │
│  장바구니 패널             →  생성 중 오버레이                   │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 상세 매핑 분석

### 3.1 랜딩 페이지 ← Bécane 메인

**참고 스크린샷**: `main-collection.png`, `main-hover.png`

**Bécane 메인 특징**:
- 대형 타이틀 "COLLECTION" (3rem+)
- 11명 모델 Chronophotography (수평 일렬)
- 하단 플로팅 바 "14 PRODUCTS" + "DISCOVER"
- 순백 배경, 블랙 타이포
- 극도의 여백

**의뢰소 적용**:
```
┌─────────────────────────────────────────────────────────────────┐
│  의뢰소              ALL 00  STORIES 00         CART 00        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  REQUEST LAB 01 / 01                                           │
│  의뢰소                                    ← 대형 한글 타이틀    │
│  MEMBERS  5                                                     │
│                                                                 │
│  👤 👤 👤 👤 👤  (5명 멤버 Chronophotography)                   │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  5 MEMBERS                                      [의뢰 시작]     │
└─────────────────────────────────────────────────────────────────┘
```

**핵심 변경**:
- 어두운 배경 → 순백 배경
- 빨간 CTA 버튼 → 플로팅 바 안 "의뢰 시작"
- 기존 히어로 비디오 → 멤버 Chronophotography (이미지/호버 시 동영상)

## 4. 공통 컴포넌트 정의

### 4.1 헤더 (Global Header)

```
┌─────────────────────────────────────────────────────────────────┐
│  의뢰소        ALL 00  STORIES 00         CART/CLOSE/NEW 00    │
└─────────────────────────────────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 의뢰소 | 로고, 클릭 시 홈 |
| ALL 00 | 전체 생성 기록 수 (선택) |
| STORIES 00 | 스토리/튜토리얼 (선택) |
| 우측 | 상황별 변경 (CART/CLOSE/NEW) |

### 4.2 플로팅 바 (Bottom Bar)

```
┌─────────────────────────────────────────────────────────────────┐
│  [좌측 정보/필터]                            [우측 CTA 버튼]    │
└─────────────────────────────────────────────────────────────────┘
```

**사용처**:
- 랜딩: "5 MEMBERS" + "의뢰 시작"
- 멤버 선택: "LUNA 선택" + "다음"
- 음악 선택: "Aurora Night" + "다음"
- 모션 선택: "[필터 탭들]" + "Catwalk [생성]"
- 결과: "[다운로드] [공유]" + "[다시 만들기]"

### 4.3 스텝 인디케이터

```
STEP 1 / 4   또는   MEMBER 01 / 05
```

- 우측 상단 고정
- 작은 텍스트, 회색
- 진행 상황 표시

## 5. 디자인 시스템 적용

### 5.1 컬러 (Bécane 스타일 적용)

| 용도 | 기존 | 변경 |
|------|------|------|
| 배경 | #1A1A1A (어두움) | #FFFFFF (순백) |
| 주 텍스트 | #FFFFFF | #000000 |
| 보조 텍스트 | #555555 | #888888 |
| CTA | #D4231A (강한 빨강) | #000000 (블랙) 또는 최소한의 빨강 |
| 선택 상태 | 빨간 테두리 | 얇은 블랙 테두리 |

### 5.2 타이포그래피

| 요소 | 스타일 |
|------|--------|
| 대형 타이틀 | 3rem+, Extra Bold, 대문자 |
| 서브타이틀 | 0.75rem, Regular, 대문자, 회색 |
| 본문 | 1rem, Regular |
| 버튼 | 0.875rem, Medium, 대문자 |

### 5.3 여백

| 위치 | 값 |
|------|-----|
| 섹션 간 | 80px+ |
| 페이지 좌우 | 24px (모바일), 48px+ (데스크톱) |
| 컴포넌트 간 | 24px |

## 6. 우선순위 및 구현 순서

### Phase 3.1: 기초 리빌드
1. ✅ 공통 레이아웃 컴포넌트 (헤더, 플로팅 바)
2. ✅ 디자인 토큰 (컬러, 타이포, 여백)

4. 멤버 선택 (Look 상세 스타일 또는 랜딩 통합)
5. 음악 선택 (Stories 갤러리 스타일)
6. 모션 선택 (ALL 그리드 스타일)

### Phase 3.3: 생성/결과
7. 생성 중 오버레이 (장바구니 패널 스타일)
8. 결과 페이지 (Look 상세 스타일)
